Threat Modelling Approach

The threat modelling process identifies potential attack vectors, evaluates their likelihood and impact, and informs the design of detection and mitigation strategies. It draws on the vulnerability taxonomy established in the research phase.

Detection & Evaluation Logic

The detection system combines multiple analysis techniques to identify injection attempts with high accuracy while minimising false positives. The evaluation logic then determines the appropriate response based on the severity and type of detected threat.

Detection Pipeline Flowchart Detailed flowchart of the detection and evaluation pipeline will be added here
  • Pattern matching: Rule-based detection of known injection signatures and templates.
  • Semantic analysis: Contextual understanding of prompt intent beyond surface-level patterns.
  • Anomaly detection: Identification of statistically unusual prompt structures or content.
  • Ensemble scoring: Aggregation of multiple detection signals into a unified risk score.

Limitations

Important: No detection algorithm can guarantee 100% accuracy against prompt injection. The evolving nature of adversarial techniques means continuous evaluation and updates are necessary.

  • Evasion risk: Sophisticated obfuscation techniques may bypass pattern-based detection.
  • False positives: Legitimate prompts with unusual structure may be flagged incorrectly.
  • Context dependence: Detection accuracy varies based on application domain and model configuration.
  • Computational cost: Multi-stage analysis adds latency that may affect real-time applications.
