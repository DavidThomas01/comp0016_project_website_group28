Prompt Injection: Vulnerabilities, Mitigations, and Testing

This section summarises the research conducted by Team 28 into prompt injection attacks, drawing directly from our report produced in collaboration with Avanade. It provides the theoretical and empirical foundation for the platform design.

Introduction and Background

The rapid adoption of large language models (LLMs) in enterprise systems has introduced new security challenges. Unlike traditional software, LLMs interpret all inputs as natural language, making it difficult to distinguish between trusted instructions and untrusted data.

Prompt injection exploits this ambiguity by embedding adversarial instructions into user input, retrieved content, or tool outputs, potentially causing models to bypass safeguards or perform unintended actions. This research focuses on understanding these attacks and their implications for real-world AI deployments, particularly in enterprise contexts. :contentReference[oaicite:1]{index=1}

Prompt Injection Vulnerabilities

Prompt injection vulnerabilities arise from fundamental characteristics of LLMs, rather than from conventional implementation bugs. Our research identifies several major classes of vulnerabilities observed in modern AI systems. :contentReference[oaicite:2]{index=2}

  • Direct Prompt Injection: Explicit attempts by users to override system instructions (e.g. “ignore previous instructions”).
  • Indirect Prompt Injection: Malicious instructions embedded in external data such as documents, webpages, or emails.
  • Instruction–Data Boundary Confusion: Lack of a strict separation between instructions and data within the model context.
  • Obfuscation and Encoding Attacks: Use of Base64, Unicode homoglyphs, or multilingual prompts to bypass filters.
  • Tool-Use and Action Injection: Manipulation of tool calls or API arguments generated by the model.
  • Memory and Long-Horizon Vulnerabilities: Gradual injection across multiple turns leading to behavioural drift.
  • Multimodal Prompt Injection: Hidden instructions embedded in images or other non-text modalities.
  • Multi-Agent Contamination: Propagation of malicious instructions across agent workflows.

Mitigation Strategies

Our research shows that no single mitigation can fully eliminate prompt injection. Instead, effective defence requires a layered, defence-in-depth approach combining multiple complementary strategies. :contentReference[oaicite:3]{index=3}

  • Input Validation and Sanitisation: Treating all inputs as untrusted and filtering malicious instructions before model execution.
  • Robust Prompt Design: Structuring prompts to clearly separate trusted instructions from untrusted data.
  • Output Filtering and Guardrails: Validating model outputs before returning them or executing actions.
  • Multi-Agent Defence Pipelines: Distributing responsibility across specialised agents to reduce single-point failures.
  • Context Sanitisation: Analysing and filtering long or externally sourced context before use.
  • Sandboxing and Permission Control: Restricting tool access and enforcing strict execution boundaries.
  • Model-Level Alignment: Improving baseline robustness through fine-tuning and alignment techniques.

Testing Against Prompt Injection

Testing is essential for identifying prompt injection vulnerabilities before deployment. Our research surveys both academic and industry testing methodologies designed to probe the full attack surface of modern AI systems. :contentReference[oaicite:4]{index=4}

Testing Approaches

  • Attacker-model testing using adversarial LLMs.
  • Evolutionary search to generate novel adversarial prompts.
  • Black-box and white-box testing methodologies.
  • Red-team templates for social and behavioural manipulation.
  • Testing of RAG pipelines, tool interfaces, and MCP servers.
  • Long-horizon and memory-persistence testing.

Testing Frameworks

The report evaluates several established frameworks, including NVIDIA garak, Microsoft PromptBench and PyRIT, OWASP PromptMe, and OpenAI Evals. These tools enable systematic benchmarking, adversarial testing, and integration into CI/CD pipelines. :contentReference[oaicite:5]{index=5}

Key Research Findings

  • Modern LLMs are relatively robust to simple direct injections.
  • Obfuscated and indirect prompt injection remains significantly harder to defend against.
  • Tool-mediated and MCP-based systems substantially expand the attack surface.
  • Prompt injection is primarily an architectural and operational challenge rather than a model-selection problem.

These findings directly inform the design decisions and evaluation criteria used in the Prompt Injection Protection platform. :contentReference[oaicite:6]{index=6}
