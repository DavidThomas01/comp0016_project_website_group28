Prompt Injection: Vulnerabilities, Mitigations & Testing

A comprehensive summary of the research conducted by Team 28 into prompt injection attacks, forming the theoretical and empirical foundation for the platform design.

Introduction and Background

The rapid adoption of large language models (LLMs) in enterprise systems has introduced new security challenges. Unlike traditional software, LLMs interpret all inputs as natural language, making it difficult to distinguish between trusted instructions and untrusted data.

Key insight: Prompt injection exploits the fundamental ambiguity between instructions and data in LLM systems — embedding adversarial instructions into user input, retrieved content, or tool outputs to cause unintended actions.

Prompt Injection Vulnerabilities

Prompt injection vulnerabilities arise from fundamental characteristics of LLMs, rather than from conventional implementation bugs. Our research identifies several major classes of vulnerabilities observed in modern AI systems.

Direct prompt injection

Explicit attempts by users to override system instructions (e.g. "ignore previous instructions").

User Input

Indirect prompt injection

Malicious instructions embedded in external data such as documents, webpages, or emails.

External Data

Instruction–data confusion

Lack of strict separation between instructions and data within the model context.

Architecture

Obfuscation attacks

Use of Base64, Unicode homoglyphs, or multilingual prompts to bypass input filters.

Evasion

Tool-use injection

Manipulation of tool calls or API arguments generated by the model.

MCP Relevant

Memory vulnerabilities

Gradual injection across multiple turns leading to behavioural drift.

Long-Horizon

Multimodal injection

Hidden instructions embedded in images or other non-text modalities.

Cross-Modal

Multi-agent contamination

Propagation of malicious instructions across interconnected agent workflows.

Multi-Agent

Mitigation Strategies

Defence in depth: No single mitigation can fully eliminate prompt injection. Effective defence requires a layered approach combining multiple complementary strategies.

Input Validation & Sanitisation

Treating all inputs as untrusted and filtering malicious instructions before model execution.

Robust Prompt Design

Structuring prompts to clearly separate trusted instructions from untrusted data.

Output Filtering & Guardrails

Validating model outputs before returning them or executing downstream actions.

Multi-Agent Defence

Distributing responsibility across specialised agents to reduce single-point failures.

Context Sanitisation

Analysing and filtering long or externally sourced context before use.

Sandboxing & Permission Control

Restricting tool access and enforcing strict execution boundaries.

Testing Against Prompt Injection

Testing is essential for identifying prompt injection vulnerabilities before deployment. Our research surveys both academic and industry testing methodologies designed to probe the full attack surface of modern AI systems.

Testing Approaches

  • Attacker-model testing using adversarial LLMs.
  • Evolutionary search to generate novel adversarial prompts.
  • Black-box and white-box testing methodologies.
  • Red-team templates for social and behavioural manipulation.
  • Testing of RAG pipelines, tool interfaces, and MCP servers.
  • Long-horizon and memory-persistence testing.

Testing Frameworks

NVIDIA Garak

Open-source LLM vulnerability scanner with probe-based testing.

Microsoft PyRIT

Python Risk Identification Tool for generative AI red-teaming.

PromptBench

Unified benchmarking library for evaluating adversarial prompt robustness.

OWASP PromptMe

Framework aligned with OWASP Top 10 for LLM Applications.

OpenAI Evals

Evaluation framework for systematic model behaviour assessment.

Custom Pipelines

Project-specific test suites targeting MCP server attack surfaces.

Key Research Findings

Simple Injections Declining

Modern LLMs are relatively robust to simple direct injections thanks to improved alignment and training.

Obfuscation Remains Dangerous

Obfuscated and indirect prompt injection remains significantly harder to defend against.

MCP Expands Attack Surface

Tool-mediated and MCP-based systems substantially expand the potential attack surface for injection.

Architectural Challenge

Prompt injection is primarily an architectural and operational challenge rather than a model-selection problem.

These findings directly inform the design decisions and evaluation criteria used in the Prompt Injection Protection platform.
