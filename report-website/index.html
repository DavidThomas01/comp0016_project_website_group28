Project Overview

Why prompt injection matters

Prompt injection is an emerging class of vulnerability where adversarial instructions buried in user input or external data cause an LLM to ignore its original task, leak sensitive information, or misuse attached tools.

This risk is especially acute in agentic systems and MCP-style architectures, where models act as orchestrators of tools, APIs, and external services and can therefore cause real-world side effects when compromised.

What this platform provides

  • An accessible catalogue of prompt injection vulnerabilities and mitigation strategies.
  • Technical explanations, worked attack examples, and side-by-side “with vs without mitigation” demos.
  • A testing environment for defining prompts, choosing models and mitigations, and running structured experiments.
  • A self-contained lab for learning about, experimenting with, and evaluating defences in tool-using LLM systems.

Problem Statement

The security risk

Prompt injection attacks occur when malicious instructions are smuggled into the inputs or context seen by an LLM, causing it to override its original instructions, reveal hidden information, or execute actions it should refuse.

In systems where the model can call tools, access documents, or route requests via MCP-style servers, a single successful injection can lead to data exfiltration, incorrect tool calls, or unsafe changes in downstream systems.

Gaps we identified

  • Few resources connect high-level theory to concrete, step-by-step prompt injection attack scenarios.
  • Existing testing approaches are often bespoke scripts that are hard to compare, reuse, or track over time.
  • Developers lack visual support for seeing where mitigations are applied and how they can fail.
  • Teams struggle to harden system prompts without accidentally changing behaviour or introducing new risks.

Our main stakeholders are developers, security engineers, and organisations building tool-using LLM applications, along with students and researchers who need a practical environment to study these attacks.

Our Platform

Full-stack lab

A full-stack web platform combining an interactive frontend with a FastAPI backend and persistent storage, centred around a vulnerability catalogue that introduces different classes of prompt injection attacks with concise summaries and deep-dive views.

Exploration views

Vulnerability and mitigation pages explain each attack, show realistic prompts and “bad responses”, and present recommended defences with clear “with vs without mitigation” flows.

Hands-on tooling

A testing workbench, prompt enhancer, and security knowledge assistant let users configure experiments, harden their own system prompts, and reason about attack surfaces and mitigation choices in one place.

Context

⚠️ Prompt Injection Attacks

Prompt injection occurs when malicious instructions are embedded within user inputs to trick an AI model into ignoring intended instructions or revealing information it should not. Examples include extracting system prompts, redirecting actions to malicious endpoints, or bypassing content and access controls.

🔗 Model Context Protocol (MCP)

MCP standardises how AI models access external tools, APIs, and knowledge sources. MCP servers mediate access between the model and external systems — making them a critical surface for security evaluation, especially in public or enterprise deployments.

Key Features & Future Directions

The platform provides a set of concrete tools for exploring and defending against prompt injection today, while also opening space for future extensions and deeper integrations.

Current Key Features

  • Explore real-world prompt injection vulnerabilities through an interactive catalogue with explanations, example attack prompts, and “what went wrong” narratives.
  • Dive into mitigation strategies with dedicated pages that explain defensive patterns and show implementation-style snippets in pseudo-code, Python, and Java.
  • Run structured tests against LLMs in a testing workbench where you can configure prompts, models, and mitigations, then review saved results in one place.
  • See layered defences in action as the backend applies pre-input checks, prompt-level guidelines, post-output filtering, and monitoring when running each test.
  • Automatically evaluate injection success using an analyser model that flags likely successful attacks, assigns a risk score, and explains its reasoning.
  • Harden your own system prompts with a three-stage prompt enhancer that improves wording, prepends selected mitigations, and verifies that intent is preserved.
  • Ask a security knowledge assistant questions about prompt injection and mitigations, with responses grounded in the platform’s curated knowledge base.
  • Work with multiple LLM providers via a pluggable backend that routes requests through a registry of OpenAI, Anthropic, and compatible models.

Future Directions

  • Integrate with MCP servers and tool registries so tests can directly target tool-using agents and evaluate how prompt injection affects real tool calls.
  • Add richer visual analytics for test runs, such as timelines or risk heatmaps that highlight which prompts, models, or mitigation combinations fail most often.
  • Support collaborative test suites, allowing teams to share, tag, and version prompt injection tests across projects and environments.
  • Introduce scenario-based “attack campaigns” that chain multiple prompts and contexts to simulate complex, multi-step injection attempts.
  • Extend the mitigation catalogue with more language-specific code examples and ready-to-adapt middleware patterns for common frameworks.
  • Export reports in richer formats, such as branded PDF or HTML summaries that organisations can attach to internal security reviews.
  • Provide guided “wizards” for new users, walking them through defining their first tests, choosing mitigations, and interpreting the analyser’s output.
  • Add plug-in runners for external LLM gateways, enabling side-by-side comparison of different model providers or hosted agent platforms under the same tests.
