Exploring vulnerabilities, mitigations, and testing methodologies for AI-driven applications — with a focus on securing Model Context Protocol (MCP) servers against prompt injection attacks.

Project Overview

Prompt injection is an emerging attack vector where malicious instructions are embedded in user inputs to influence an AI model's behaviour — potentially bypassing safeguards or causing the model to expose sensitive information.

Why it matters: As organisations connect LLMs to tools, APIs, and knowledge sources via protocols like MCP, the attack surface expands significantly. Our project maps these risks, evaluates defences, and builds testing tools to help secure these systems.

Working in collaboration with Avanade and supervised by Dr Yun Fu at UCL Computer Science, Team 28 is building a comprehensive platform for understanding and combating prompt injection threats.

Objectives

🛡️

Evaluate Prompt Injection Techniques

Document current prompt injection attack vectors and assess their real-world impact on AI systems and enterprise deployments.

🔍

Assess MCP Vulnerabilities

Investigate AI-application-specific vulnerabilities with particular emphasis on Model Context Protocol servers and tool-connected systems.

🧪

📋

Deliver actionable recommendations and tooling approaches to help organisations secure their MCP servers against these threats.

Context

⚠️ Prompt Injection Attacks

Prompt injection occurs when malicious instructions are embedded within user inputs to trick an AI model into ignoring intended instructions or revealing information it should not. Examples include extracting system prompts, redirecting actions to malicious endpoints, or bypassing content and access controls.

Direct Injection Indirect Injection Obfuscation Tool-Use Attacks

🔗 Model Context Protocol (MCP)

MCP standardises how AI models access external tools, APIs, and knowledge sources. MCP servers mediate access between the model and external systems — making them a critical surface for security evaluation, especially in public or enterprise deployments.
