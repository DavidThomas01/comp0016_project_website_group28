Prompt Injection
Protection

This project delivers an educational web platform for understanding, exploring, and testing prompt injection attacks against large language models, with a particular focus on tool-using and MCP-style systems.

View Requirements → Read Research

Project abstract

Problem and motivation

Large language models can be manipulated through prompt-injection attacks. Hidden instructions in user input or other context can make a model ignore its task, leak data, or misuse tools.

That risk is especially serious when the model calls APIs or MCP-style servers, because failures can propagate into real systems and client environments. There is still a gap between theory and clear, repeatable attack examples.

Teams need practical ways to learn how attacks work, what mitigations exist, how to run tests, compare mitigations, and harden system prompts without breaking intended behaviour. We treat prompt-injection risk as something to study systematically—through structured content on vulnerabilities and mitigations, and through tooling that runs prompts against real models and inspects the results.

Platform and technical approach

We built an educational web application with a React single-page front end and a FastAPI back end backed by SQLite. The platform presents a vulnerability catalogue with detail pages, interactive demos, knowledge-check quizzes, and links to mitigations.

Mitigation guides give structured explanations and optional code-style snippets. A testing workbench lets users configure and run tests against routed LLM providers. A three-stage prompt-enhancement pipeline (improve, prepend mitigation rules, verify) and a streaming security-assistant chat, with keyword search over the same catalogue, round out the user-facing surface.

The backend centralises model access through a provider router, applies configurable mitigation layers around test execution where appropriate, and analyses model outputs for injection-related risk.

Outcomes and intended impact

Developed in collaboration with Avanade, the platform supports technical and non-technical users in understanding prompt injection and reducing risk when delivering client solutions.

It combines structured learning with interactive testing: teams can study the material and run realistic checks so that client work can be made more secure, and Avanade employees can learn and practise alongside their day-to-day work with AI systems.

Platform features

Vulnerabilities

This is the prompt-injection vulnerability catalogue. The home view lists every vulnerability with search and impact filters. Opening one shows the full article: explanation, tags, impact, example attack, demo (attack vs mitigated chat), links to mitigations, optional quiz, previous/next navigation, and export to PDF.

  • Search and filter by impact (High / Medium / Low).
  • Open a card for the full page; run the step-by-step demo and optional quiz.
  • Jump to related mitigations and export a PDF for the current vulnerability.
Testing page

The “Test Prompt Injection” area is where you build tests, save setups, and run them against external models. You pick or create a test, set the model, system prompt, mitigations, and how to call the API (including OpenAI-style, Anthropic, or a custom endpoint). You can check the connection, run a normal test, or run a Garak scan. Results show in chat, analysis, and framework history; tests and saved configs are stored on the backend.

  • Create, edit, duplicate, and delete tests; load and save named configurations.
  • Validate the API connection, then run a test or a Garak scan.
  • Review analysis and scan history; export a test report PDF where supported.
Mitigations

The mitigations section explains defence techniques for prompt injection. A list on the left shows every mitigation with a short line and a code vs non-code badge. The main panel gives strategy, typical flow, and implementation—either a code snippet (with language choice and copy) or a note that it is not a single code module. You can export a PDF and choose which languages to include.

  • Browse the list and pick a mitigation to read the full detail.
  • Switch snippet language, copy code, and export PDF with optional language choices.
Security Knowledge Assistant (chatbot)

The “Security Knowledge Assistant” is a floating chat button on every page, plus a full-screen route for longer conversations. It answers questions about prompt injection, mitigations, and testing, shows a welcome message with suggested questions, streams replies as Markdown, and lets you stop or clear the conversation. The thread is saved in the browser so you can come back later.

  • Open or close the panel from the corner, or use the dedicated full-page view.
  • Send messages, use suggested questions, stop generation, or clear the chat.
Prompt enhancer

The Prompt Enhancer helps you harden a system prompt. You choose prompt-layer mitigations from a list, paste your system prompt, pick a model, and run Generate. The app restructures the prompt, prepends mitigations, and runs checks. The result panel shows original, improved, and enhanced text plus verification (pass/fail style notes and which mitigations were applied). You can copy or reset.

  • Toggle mitigations, select a model, paste your prompt, then Generate or Reset.
  • Copy any of the outputs when you are happy with the result.
