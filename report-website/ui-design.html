UI Design

The user interface design process, from initial design goals and user personas through to wireframes, prototyping, and accessibility-first development.

Design Goals

Coming soon: Detailed design rationale and visual design principles will be documented here once the design system is finalised.

The UI design is guided by a set of core principles that prioritise clarity, usability, and educational value. The platform must serve both technical security professionals and semi-technical users exploring prompt injection for the first time.

Clarity First

Complex security concepts presented through clean, intuitive interfaces with clear visual hierarchy.

Guided Exploration

Users can navigate freely between vulnerabilities, mitigations, and testing tools with logical flow.

Accessible by Default

WCAG AA compliance, keyboard navigation, screen reader support, and high-contrast mode built in.

Personas & User Needs

User personas were developed through stakeholder interviews and analysis of the target audience to ensure the platform meets the needs of its primary user groups.

Security Engineer

Experienced professional evaluating MCP server deployments. Needs detailed technical analysis, advanced testing tools, and exportable reports.

Technical Testing Focus

CS Student / Researcher

Learning about AI security. Needs clear explanations, working examples, and educational walkthroughs of attack techniques and defences.

Educational Examples Focus

Technical Manager

Overseeing AI integration. Needs high-level risk summaries, mitigation guidance, and compliance-aligned reporting.

Strategic Compliance

AI Developer

Building applications with LLMs. Needs practical mitigation code, testing APIs, and integration guidance.

Accessibility Considerations

Accessibility is treated as a fundamental design requirement, not an afterthought. The platform is built to meet WCAG 2.1 AA standards throughout.

Keyboard Navigation

All interactive elements accessible via keyboard with visible focus indicators and logical tab order.

Responsive Design

Fully responsive layouts tested across mobile, tablet, and desktop viewport sizes.

Colour & Contrast

Colour is never the sole indicator of meaning; all text meets WCAG AA contrast ratios.

Screen Reader Support

Semantic HTML, ARIA labels, and meaningful alt text for full screen reader compatibility.
