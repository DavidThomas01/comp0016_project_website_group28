Evaluation & Testing

The evaluation strategy, test infrastructure, key metrics, and results from comprehensive testing of the platform against prompt injection vulnerabilities.

Summary of Achievements

The platform has been developed against the MoSCoW-prioritised requirements for the Prompt Injection Protection system (Systems Engineering, IXN, Group 28, Avanade). Below is a structured summary of how the delivered system meets the must-have functional and non-functional requirements, written for easy traceability and reading.

Functional — Must Have

The system delivers the following core features as specified.

Home Page

  • A dedicated home page that introduces prompt injection, associated risks, and a high-level explanation of how prompt injection works.
  • A list of known prompt injection vulnerabilities, each clickable with a short summary.

In-Depth Vulnerability View

  • Selecting a vulnerability from the home page navigates to a detailed view.
  • An information panel with a technical explanation of the vulnerability.
  • An example panel showing how a model behaves when faced with that vulnerability.
  • A clear explanation of at least one mitigation strategy per vulnerability.
  • A comparison panel showing model behaviour without mitigation vs. with mitigation applied.

Knowledge Base

  • Clear examples of prompt injection vulnerabilities with technical explanations.
  • Working attack examples for each vulnerability type.
  • Clear explanations of prompt injection mitigations with technical detail for each.

Testing View

  • Accessible from the main navigation bar.
  • Users can create custom tests for individual mitigations, with tests named by the user and stored in memory for the session.
  • A chat-style interface that processes a single prompt; users can enable or disable mitigations (one per test) identified during research.
  • Tests show a clear pass/fail result based on predefined security expectations.
  • Test data is preserved locally in an SQL database; all tests run on a fixed model (GPT 5.0).

Non-Functional — Must Have

The system meets the following quality and delivery criteria.

Delivery & Form

  • The system is delivered as a web application, usable in standard browsers without a persistent backend server requirement.

Clarity & Audience

  • Vulnerabilities, mitigations, and test results are presented in a clear and understandable manner for both technical and semi-technical users.

Performance

  • The chatbot and testing interfaces provide acceptable interactive response times for typical use.

Interface & Standards

  • An intuitive, clear, and logical interface that conforms to agreed standards and supports consistent navigation across views.

Accessibility & Preference

  • The system supports a dark mode interface so users can choose a preferred visual theme.

Evaluation Strategy

Coming soon: Detailed evaluation results and analysis will be added as testing progresses. The strategy and metrics framework are defined below.

The evaluation strategy covers functional correctness, security effectiveness, usability, and performance. Testing is structured in layers — from unit tests of individual components through to end-to-end integration testing of the full platform against real prompt injection scenarios.

🧱

Unit Tests

Individual component and function-level testing.

🔗

Integration

Cross-component interaction and data flow validation.

🏗️

System

End-to-end testing of complete user workflows.

👤

Acceptance

User acceptance testing with stakeholders and target users.

Test Setup

The test environment is designed to provide reproducible, isolated conditions for evaluating the platform's detection and mitigation capabilities.

🧪 Test Environment

Isolated testing environment with controlled access to LLM APIs and MCP server instances. All tests run against consistent model configurations to ensure reproducibility.

📁 Test Data

Curated dataset of prompt injection attacks spanning all vulnerability categories identified in the research phase, including both known and novel attack variants.

⚙️ Configuration

Standardised test configurations with documented model parameters, mitigation settings, and evaluation criteria for each test category.

🔄 Automation

Automated test execution pipelines that can re-run the full test suite on demand, enabling regression testing and comparative evaluation.
