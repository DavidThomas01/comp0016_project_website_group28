System Design

The high-level architecture and system design of the platform, including component overview, data flow patterns, technology stack, and security architecture.

High-Level Architecture

Coming soon: The full architecture diagram and detailed component descriptions will be added once the system design is finalised.

The platform follows a modular architecture that separates concerns between the user interface, business logic, data management, and external integrations. This design enables independent development and testing of each component.

🏗️ System Architecture Diagram High-level architecture diagram showing all major components and their interactions

Component Overview

The system is composed of several key components, each responsible for a distinct aspect of the platform's functionality.

🖥️

Frontend Application

Web-based interface providing the vulnerability explorer, knowledge base, and testing views. Delivers responsive, accessible UI across devices.

UI Layer Responsive
⚙️

Analysis Engine

Core processing logic for prompt analysis, vulnerability detection, and mitigation evaluation. Implements the algorithmic pipeline.

Core Logic Processing
🧪

Testing Framework

Automated testing infrastructure for running prompt injection tests against MCP servers with configurable mitigation strategies.

Testing Automation
💾

Data Layer

Local SQL database for preserving test data, knowledge base content, and user configurations. No persistent backend server required.

Local Storage SQL

Data Flow

Data flows through the system in a well-defined pipeline, from user input through analysis and testing to results presentation.

🔄 Data Flow Diagram Sequence diagram showing how data moves between components during key operations

1. Input

User provides prompts, selects vulnerabilities, or configures tests via the frontend.

2. Process

The analysis engine evaluates inputs, applies detection logic, and runs mitigations.

3. Output

Results are formatted and presented with clear pass/fail indicators and detailed analysis.

Technology Stack

Technology choices are driven by the project requirements: web delivery, local-first data, and no persistent backend server.

Layer Technology Rationale
Frontend To be confirmed Responsive web UI meeting accessibility requirements
Logic To be confirmed Analysis engine and detection pipeline
Data SQL (local) Local-first storage per non-functional requirements
LLM Integration GPT-5.0 (fixed) Standardised model as specified in requirements
Protocol MCP Target protocol for security testing and evaluation

Security Architecture

Security by design: Given that the platform analyses security vulnerabilities, its own security architecture is critical. Defence-in-depth principles are applied throughout.

🔒

Sandboxed Execution

All prompt testing runs in isolated environments with strict resource and permission controls.

🛡️

Input Boundaries

Clear separation between user-provided content and system instructions at every layer.

📝

Audit Logging

Comprehensive logging of all test operations and analysis results for traceability.

🔐

Minimal Permissions

Principle of least privilege applied to all tool and API interactions.
