Evaluation

How we assessed the delivered platform against the requirements, the team’s contributions, and a critical look at what worked and what did not.

Summary of Achievements

The tables in Requirements traceability are the source of truth; below is a short recap you can expand if needed.

Functional — Must have

Home page, vulnerability catalogue with in-depth views (examples, mitigations, comparison), structured knowledge content, and a full testing workbench — delivered as specified.

See a fuller summary

Home

  • Introduces prompt injection, risks, and how attacks work; lists vulnerabilities with short summaries and navigation into detail.

Vulnerability view

  • Technical explanation, behaviour example, mitigation(s), and a comparison of unmitigated vs mitigated model output.

Knowledge base

  • Worked examples, attack demonstrations, and mitigation explanations with enough depth to support learning and testing.

Testing

  • Custom named tests, chat-style runs with per-test mitigations, pass/fail expectations, server-side SQL storage, default platform models in the GPT-5.x family (see evaluation notes for naming vs “GPT 5.0” in the spec).

Non-functional — Must have

Browser-based delivery, clear presentation for technical and semi-technical readers, responsive interactive areas, consistent standards-aligned UI, and dark mode.

See a fuller summary

Delivery & form

  • Web application usable in standard browsers without requiring a persistent backend for basic use.

Clarity & audience

  • Vulnerabilities, mitigations, and test outcomes are explained so both technical and semi-technical users can follow them.

Performance & UI

  • Chat and testing flows stay acceptably responsive; layout and navigation follow agreed interface patterns.

Preference

  • Dark mode is available as a first-class theme choice.

Known bugs

We tested the main parts of the product carefully and fixed the issues we found. At this point, we are not aware of any major bugs in the current version.

See details

There were some challenges during development, but we worked through them and made sure the key features work as expected. We tested the most important flows from start to finish.

We believe it is a solid base for Avanade to build on. We also included documentation to make sure anyone at Avanade can understand the platform, maintain it, and add new functionalities in the future.

We are very excited about the positive feedback from Avanade and Microsoft, and about the article we are writing for their blog (see our development blog) and the presentation we will be giving on their AI Foundry Discord.
